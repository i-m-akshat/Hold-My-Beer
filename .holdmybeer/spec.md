# HoldMyBeer — Project Model Specification (spec.md)

> Generated by `hmb-crack` (OP: INPUT→PSM.domain). Source of truth: `.holdmybeer/psm.json`.

## Context

The toolkit currently installs (and the `hmb` init CLI runs), after which the PowerShell / terminal
window **closes immediately** once the script finishes — leaving the user with no readable confirmation
of what happened. The user wants a **beautiful, proper completion message** that stays on screen.

This spec captures that single requested capability as **one Feature** implementing **two Requirements**
(clear messaging + no abrupt window close).

### Non-Goals
- Does **not** change *what* gets installed (skills, commands, shared/) or the supported platforms
  (Claude, Gemini, Codex, Cursor).
- Does **not** change install paths, the `--force`/`-Force` behavior, or the skip logic.
- Exact visual theme/branding of the message is open to later refinement; the requirement is
  *readable, formatted, and persistent* output.

## Requirements

### `req-install-message` — Display a proper completion message
- **Status:** proposed
- **Priority:** high
- **Confidence:** 0.85 (source: user)
  - *Reason:* User explicitly wants a "beautiful and proper message" rather than the window closing
    with no readable output; intent is unambiguous.
- **Acceptance Criteria:**
  1. A formatted summary is printed showing installed items, skipped items, and next-step instructions on success.
  2. On failure or error, a clear human-readable error message is shown (not just a stack trace or silent window close).

### `req-install-no-abrupt-close` — Do not close the terminal abruptly
- **Status:** proposed
- **Priority:** high
- **Confidence:** 0.80 (source: user)
  - *Reason:* User states the window "immediately closing" is undesirable and wants it to stay open with a message.
- **Acceptance Criteria:**
  1. After printing the summary, the script waits for user input (e.g., "Press any key to exit") before the window closes.
  2. Behavior applies to `install.ps1`/`install.sh` and the `hmb` init CLI (`hmb.cmd`/`hmb.ps1`) when launched by double-click.

## Features

### `feat-install-summary` — Polished post-run summary + persistent terminal
- **Status:** proposed
- **Actor:** End user installing or initializing the toolkit
- **Implements:** `req-install-message`, `req-install-no-abrupt-close`
- **Confidence:** 0.80 (source: user)
  - *Reason:* Directly requested as a single feature covering messaging + terminal persistence.
    Visual styling ("beautiful") is subjective and may be refined during brew.
- **Acceptance Criteria:**
  1. A formatted success/failure summary is displayed after install or hmb init.
  2. The terminal pauses for user acknowledgment and does not close abruptly.
