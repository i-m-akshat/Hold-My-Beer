Getting Started Workflow
  1. Modify templates: Edit `.holdmybeer/spec.md` to document your epic or business requirements.
  2. Audit and Design: Generate your implementation blueprint using `/holdmybeer-plan` and implement safely with `/holdmybeer-code`.
  3. Final Review: Execute `/holdmybeer-review` to verify that all specifications are fully met and passes test validation.

HoldMyBeer Commands Directory
/holdmybeer-init Check for existing SDD structures or initialize templates.
/holdmybeer-spec Brews and validates specification from an Epic into spec.md.
/holdmybeer-plan Converts spec into a phased roadmap and task blueprint.md.
/holdmybeer-code Implements code changes step-by-step from blueprint.md.
/holdmybeer-review Performs pre-merge quality review check.
